# 📋 СЛАЙД 1: ВВЕДЕНИЕ

## Проблема
- ❌ Кондитеры используют ручное управление заказами через мессенджеры
- ❌ Нет системы управления каталогом
- ❌ Сложно собирать отзывы и отслеживать заказы
- ❌ Неэффективное использование времени

## Решение
✅ **Создан Telegram-бот для автоматизации процесса заказа кондитерских изделий**

## Цель проекта
Разработать полнофункциональный Telegram-бот с интегрированной базой данных для управления каталогом товаров, приёма заказов и сбора отзывов.

# 🔍 СЛАЙД 2: АНАЛИЗ АНАЛОГОВ

## Существующие решения

| Решение | Плюсы | Минусы |
|---------|-------|--------|
| **Telegram группа** | Просто | ❌ Полный хаос, потеря данных |
| **Google Forms** | Бесплатно | ❌ Неудобно, нет интеграции |
| **Готовые платформы (Shopify)** | Полный функционал | ❌ Дорого ($30-300/месяц) |
| **Собственный сайт** | Полный контроль | ❌ Высокие затраты на разработку |
| **Telegram-бот (наше решение)** | ✅ Удобно, дёшево, персонализировано | ✅ Готовое решение |

## Почему Telegram?
- 🌍 700+ млн. активных пользователей
- 📱 Не нужно разработку под iOS, Android отдельно
- 🔐 Встр��енная аутентификация через Telegram ID
- ⚡ Быстро, просто, понятно

# 🛠️ СЛАЙД 4: ТЕХНОЛОГИЧЕСКИЙ СТЕК

## Backend
- **Python 3.11+** - язык программирования
- **aiogram 3.28** - фреймворк для Telegram ботов
- **SQLAlchemy 2.0** - ORM для работы с БД
- **asyncpg 0.31** - асинхронный драйвер PostgreSQL

## Frontend
- **Inline клавиатуры** - кнопки прямо в сообщениях
- **Emoji** - красивые иконки
- **HTML разметка** - поддержка форматирования

## База данных
- **PostgreSQL** - надежная реляционная БД
- **asyncio** - асинхронная обработка

## DevOps
- **python-dotenv** - управление конфигурацией
- **Git** - версионирование кода

## Почему именно эти технологии?
✅ **Асинхронность** - один бот может обслуживать 1000+ пользователей  
✅ **ORM** - безопасность от SQL-инъекций  
✅ **PostgreSQL** - ACID гарантии, надежность  
✅ **aiogram** - лучший фреймворк для Telegram ботов на Python

# 💾 СЛАЙД 19: ПРОИЗВОДИТЕЛЬНОСТЬ И МАСШТАБИРУЕМОСТЬ

## Асинхронная обработка
```
Без асинхронности:  ❌ 1 пользователь = 1 поток (медленно)
С asyncio + aiogram: ✅ 1000+ пользователей = 1 поток (быстро)
```

## Пул соединений (Connection Pool)
```
asyncpg.create_pool(dsn, size=10)
├─ Всегда открыто 10 соединений
├─ Новые запросы берут свободное соединение
└─ Нет пересоздания подключения каждый раз
```

## Индексы в БД
```sql
CREATE INDEX idx_orders_user_id ON orders(user_id);
CREATE INDEX idx_orders_status ON orders(status);
CREATE INDEX idx_reviews_user_id ON reviews(user_id);
```

## Результаты
- ⚡ Время отклика: < 100ms
- 📊 Одновременные пользователи: 500-1000
- 🗄️ Размер БД: ~100MB

# ❓ СЛАЙД 30: ВОПРОСЫ И ОТВЕТЫ

## Возможные вопросы комиссии

**Q: Почему именно Python?**  
A: Лучшая экосистема для Telegram ботов, простой синтаксис, быстрая разработка.

**Q: Почему PostgreSQL?**  
A: ACID гарантии, надежность, поддержка JSONB, отличная интеграция с Python.

**Q: Как справиться с большим количеством пользователей?**  
A: Асинхронность + пул соединений позволяет обрабатывать 1000+ пользователей.

**Q: Как защиты от SQL-инъекций?**  
A: Параметризованные запросы ($1, $2...) - все данные экранированы.

**Q: Почему FSM для заказов?**  
A: Позволяет пользователю "вернуться" из любого шага и отслеживать состояние.

**Q: Как бот узнает, что пользователь - человек?**  
A: Через Telegram ID (невозможно подделать), аутентификация встроена в Telegram.
